## Feature Highlights

### 1. One-Command Deployment
```powershell
# Deploy everything
python deploy.py deploy --target all

# Or specific targets
python deploy.py deploy --target questions    # Data only (fastest)
python deploy.py deploy --target public       # Website only
python deploy.py deploy --target practice     # Practice app only
```

### 2. Intelligent CDN Routing
```
Browser request for KaTeX/Tailwind
  ↓
Check browser cache (instant)
  ↓
Check local cache (100-300ms)
  ↓
Try Cloudflare (if configured)
  ↓
Check CDN mirrors (jsDelivr, cdnjs)
  ↓
Fall back to original (last resort)
```

### 3. Resource Synchronization
```powershell
# Sync all resources from GitHub
python resource-sync.py sync

# Force fresh download (ignore cache)
python resource-sync.py sync --force

# Verify what's cached
python resource-sync.py verify
```

### 4. Automatic Backups
- Every deployment creates timestamped backup
- Located: `/home/ivysfyoq/backups/backup-TIMESTAMP/`
- Easy rollback if needed

### 5. Deployment Logging
- All uploads logged to `/logs/deployment.log`
- Remote status checking: `python deploy.py status`
- Complete deployment history available

## Architecture Summary

```
Personal Server (Your Machine)
  ├─ Generate questions (80k)
  ├─ Generate lesson plans
  ├─ Generate missing lessons
  └─ Run: python deploy.py deploy --target all
         │
         └─→ SSH to 192.64.118.16
             ├─ Create backup
             ├─ Upload files via SCP
             └─ Log deployment

IVY Server (192.64.118.16)
  ├─ Serve public_html/ (main website)
  ├─ Serve ib-practice-platform/ (practice app)
  ├─ Cache resources via CDN router
  └─ Fall back if external CDN unavailable

User Browser
  ├─ Load main website
  ├─ Practice platform with lazy-loaded questions
  ├─ CDN router intelligently routes assets
  └─ Cache in IndexedDB (persistent 24h)
```

## Security Notes

### Current
- Password in deploy.py (OK for development)
- Both servers in same SSH credential

### Recommended for Production
```powershell
# 1. Generate SSH keys
ssh-keygen -t ed25519 -f ~/.ssh/cpanel_key -N ""

# 2. Copy public key to server
type ~/.ssh/cpanel_key.pub | ssh ivysfyoq@192.64.118.16 "cat >> ~/.ssh/authorized_keys"

# 3. Use key in deploy.py instead of password
# 4. Store password in .env file (not in code)
# 5. Enable 2FA on cPanel account
```
