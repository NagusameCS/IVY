## Graph Algorithms: Kruskal and Dijkstra

## Overview

This lesson covers two fundamental graph algorithms: **Kruskal's algorithm** for finding minimum spanning trees and **Dijkstra's algorithm** for finding shortest paths.

## Key Concepts

- Minimum spanning trees
- Kruskal's algorithm
- Dijkstra's shortest path algorithm
- Applications of graph algorithms

### Minimum Spanning Tree

> **Important:** A **minimum spanning tree** (MST) of a weighted connected graph is a spanning tree with the smallest possible total edge weight.

A spanning tree of a graph with $n$ vertices has exactly $n - 1$ edges and connects all vertices.

### Kruskal's Algorithm

1. List all edges in order of increasing weight
2. Select the smallest edge that does **not** form a cycle with already-selected edges
3. Repeat until $n - 1$ edges are selected

**Example:** Given a graph with edges: AB(4), AC(2), BC(5), BD(10), CD(3), DE(7), CE(8)

Sorted: AC(2), CD(3), AB(4), BC(5), DE(7), CE(8), BD(10)

- Select AC(2)
- Select CD(3)
- Select AB(4)
- Skip BC(5) (forms cycle A-B-C-A)
- Select DE(7)

MST edges: AC, CD, AB, DE with total weight $2 + 3 + 4 + 7 = 16$

### Dijkstra's Algorithm

Finds the shortest path from a source vertex to all other vertices in a weighted graph (non-negative weights).

1. Set distance to source = 0, all others = $\infty$
2. Mark all vertices as unvisited
3. Select the unvisited vertex with the smallest distance
4. Update distances to its unvisited neighbours: $d(v) = \min(d(v), \; d(u) + w(u,v))$
5. Mark current vertex as visited
6. Repeat from step 3 until all vertices are visited

**Example:** Find shortest paths from A:

| Step | Visit | d(A) | d(B) | d(C) | d(D) |
|---|---|---|---|---|---|
| Init | - | 0 | $\infty$ | $\infty$ | $\infty$ |
| 1 | A | 0 | 4 | 2 | $\infty$ |
| 2 | C | 0 | 4 | 2 | 5 |
| 3 | B | 0 | 4 | 2 | 5 |
| 4 | D | 0 | 4 | 2 | 5 |

Shortest path A to D = 5 (via A-C-D).

## Summary

- Kruskal's algorithm builds an MST by greedily selecting the smallest edges that do not form cycles
- Dijkstra's algorithm finds shortest paths from a single source using a greedy approach
- Both algorithms are fundamental tools in discrete mathematics and computer science
