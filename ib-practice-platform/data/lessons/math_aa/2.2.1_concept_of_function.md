## Concept of Function

## Overview

This lesson introduces the fundamental concept of a **function** in mathematics, focusing on its definition, notation, and representation. Students will learn to identify functions from various representations and understand their key properties.

## Key Concepts

*   Understanding the definition of a function based on input and output.
*   Using function notation (f(x)) to represent relationships.
*   Distinguishing between functions and relations.
*   Evaluating functions at specific points.
*   Recognizing different representations of functions (tables, graphs, equations).

### Subtopic 1: What is a Function?

> **Important:** A **function** is a relationship between a set of inputs (domain) and a set of possible outputs (range) such that each input is associated with *exactly one* output.

A function, denoted as *f*, takes an input 'x' and produces an output 'y', which can be represented as *y = f(x)*.  It’s crucial to understand that a single input can only have one corresponding output.  This is the core defining characteristic of a function.

**Example 1:**

Consider the set of ordered pairs {(1, 2), (2, 4), (3, 6), (4, 8)}.  This represents a **relation**. However, it is *not* a function because the input '1' is associated with two different outputs (2 and 2).

**Example 2:**

Consider the set of ordered pairs {(1, 2), (2, 4), (3, 6), (4, 8)}.  This *is* a function because each input has a unique corresponding output.  We can define a function *f(x)* as: *f(x) = 2x*.

```python
def f(x):
  return 2*x
```

### Subtopic 2: Function Notation

**Function notation** provides a concise way to represent a function and its relationship between input and output.  It is written as *f(x)*, where:

*   *f* is the name of the function.
*   *x* is the input variable.
*   *f(x)* represents the output value when *x* is the input.

**Example 1:**

If *f(x) = x + 3*, then *f(2) = 2 + 3 = 5*.  This means when *x* is 2, the function *f* outputs 5.

**Example 2:**

Given the function *g(t) = t² - 1*, find *g(3)*.
*g(3) = 3² - 1 = 9 - 1 = 8*

**Exam Tip:**  Be careful to use the correct substitution rule.  Always substitute the value of the input variable into the function's equation.

### Subtopic 3: Representing Functions

Functions can be represented in various ways:

1.  **Tables:** A table lists the input values and their corresponding output values.
    *   **Note:** For a table to represent a function, each input value must have only one corresponding output value.

2.  **Graphs:** A graph plots the input and output values on a coordinate plane.
    *   **Important:** A graph represents a function if a vertical line intersects the graph at most once.

3.  **Equations:** An equation defines the relationship between the input and output variables.

**Example 1:**

Consider the function *h(x) = x²*.
| x  | h(x) = x² |
|----|----------|
| -2 | 4        |
| -1 | 1        |
| 0  | 0        |
| 1  | 1        |
| 2  | 4        |

This table represents a function because each input value has a unique corresponding output value.

**Example 2:**

The graph of *y = x + 1* is a straight line.  This represents a function because any vertical line drawn on the graph will intersect the line at exactly one point.

### Subtopic 4: Domain and Range

*   **Domain:** The **domain** of a function is the set of all possible input values (x-values) for which the function is defined.
*   **Range:** The **range** of a function is the set of all possible output values (y-values) that the function can produce.

**Example:**

For the function *f(x) = x²*, the domain is all real numbers (-∞, ∞) because you can square any real number. The range is also all non-negative real numbers [0, ∞) because the square of a real number is always non-negative.

### Subtopic 5: Identifying Functions

**Here's a test to determine if a relation is a function:**

1.  **Vertical Line Test:** If you can draw a vertical line that intersects the graph of the relation at more than one point, then the relation is *not* a function.
2.  **Check for Uniqueness:** For each input value, there must be only one corresponding output value.

**Example 1:**

Consider the relation: {(1, 2), (2, 3), (3, 4), (4, 5)}. This *is* a function.  A vertical line would intersect the graph at most once.

**Example 2:**

Consider the relation: {(1, 2), (2, 3), (3, 4), (2, 5)}. This is *not* a function. The input '2' is associated with two different outputs (3 and 5).

### Subtopic 6: Composite Functions

*   A **composite function** involves applying one function to the result of another function.

*   Notation:  If *f(x)* and *g(x)* are functions, then *f(g(x))* is the composite function.

**Example:**

Let *f(x) = x + 1* and *g(x) = x²*.  Then *f(g(x)) = f(x²)* = *x² + 1*.
Similarly, *g(f(x)) = g(x+1) = (x+1)²*.
