### Subtopic 1: Layers of the Computer System

Computer architecture is about understanding how computers *actually* work, not just using them. It's built on layers, each with a specific role, allowing for modularity and easier development.

> **Important:** The layered approach simplifies complex systems by breaking them down into manageable parts. This allows us to focus on the functionality of each layer without needing to understand the intricate details of every other layer.

The typical layers of a computer system, from bottom to top, are:

1.  **Hardware Layer:** This is the physical components – the CPU, memory, storage devices, and input/output devices. It’s the foundation of the system.
2.  **Operating System (OS) Layer:** The OS manages the hardware resources and provides a platform for applications to run. Examples include Windows, macOS, and Linux.
3.  **Application Layer:** This layer consists of the software programs we use – word processors, web browsers, games, etc.
4.  **Network Layer:**  (Relevant for networked computers) This layer handles communication between computers over a network (e.g., the Internet). It deals with addressing and routing data.

**Example:** Consider a simple task like opening a document. The hardware layer provides the physical actions, the OS manages the file system, the application loads the document, and the network layer ensures the document is sent if it’s stored on a remote server.

**Worked Example 1:**

*   **Question:** A user clicks on a web browser icon. Trace the sequence of events from the user’s action to the display of the web page.
*   **Solution:**
    1.  User clicks browser icon (Input Device - Keyboard).
    2.  OS receives the click event and activates the browser application.
    3.  Browser application sends a request to the network layer to fetch the web page.
    4.  Network layer sends the request to the web server.
    5.  Web server sends the web page data back to the network layer.
    6.  Network layer passes the data to the browser.
    7.  Browser displays the web page.

### Subtopic 2: The Central Processing Unit (CPU)

The **CPU** (Central Processing Unit) is the brain of the computer. It’s responsible for executing instructions, performing calculations, and controlling the operation of the computer. Key components of the CPU include:

*   **Arithmetic Logic Unit (ALU):** Performs arithmetic and logical operations.
*   **Control Unit:** Fetches instructions from memory, decodes them, and controls the execution of those instructions.
*   **Registers:** Small, high-speed storage locations within the CPU used to hold data and addresses during processing.

```python
# Simplified representation of a CPU cycle
# 1. Fetch: Retrieve instruction from memory
# 2. Decode: Understand the instruction
# 3. Execute: Perform the operation
# 4. Store: Write the result back to memory or a register
```

**Exam Tip:** Be able to describe the functions of the ALU, Control Unit, and Registers.

**Worked Example 2:**

*   **Question:** Explain how the ALU and Control Unit work together to perform a simple addition operation (e.g., 5 + 3).
*   **Solution:**
    1.  The Control Unit fetches the "add 5 and 3" instruction from memory.
    2.  The Control Unit sends the operands (5 and 3) to the ALU.
    3.  The ALU performs the addition operation (5 + 3 = 8).
    4.  The ALU sends the result (8) back to the Control Unit.
    5.  The Control Unit stores the result (8) in a register or memory.

### Subtopic 3: Memory – RAM and ROM

**Memory** is crucial for storing data and instructions that the CPU needs to access quickly. There are two main types:

*   **RAM (Random Access Memory):**  Volatile memory – data is lost when power is turned off. It’s used for storing data and programs that the CPU is actively using.  RAM allows for quick access to any location – hence “random access.”
*   **ROM (Read-Only Memory):** Non-volatile memory – data is retained even when power is turned off. It’s used for storing firmware and boot instructions.

| Feature          | RAM        | ROM        |
| ---------------- | ---------- | ---------- |
| Volatility       | Volatile   | Non-Volatile |
| Data Access      | Random     | Sequential |
| Usage            | Active Data | Firmware   |

**Note:**  Modern computers use a combination of RAM and secondary storage (hard drives, SSDs) for persistent data storage.

### Subtopic 4: Storage – Hard Drives and SSDs

**Storage** devices provide long-term storage for data and programs. Common types include:

*   **Hard Disk Drives (HDDs):** Mechanical devices that store data on rotating platters.  They are generally cheaper but slower than SSDs.
*   **Solid State Drives (SSDs):** Electronic devices that store data in flash memory. They are faster, more durable, and consume less power than HDDs.

### Subtopic 5: Data Representation – Binary

All data within a computer is ultimately represented in **binary** – using only 0s and 1s. This is due to the physical limitations of electronic circuits.

*   **Bits:** A single binary digit (0 or 1).
*   **Bytes:** A group of 8 bits (1 byte = 8 bits).
*   **Kilobytes (KB), Megabytes (MB), Gigabytes (GB), Terabytes (TB):** Units of data storage (1 KB = 1024 bytes, 1 MB = 1024 KB, etc.).

**Example:** The character 'A' can be represented as the decimal number 65, which is equivalent to the binary number 01000001.

### Subtopic 6: Pipelining

**Pipelining** is a technique used to improve CPU performance by overlapping the execution of multiple instructions.  Instead of waiting for one instruction to complete before starting the next, the CPU can work on multiple instructions simultaneously, like an assembly line.

> **Note:** Pipelining introduces the concept of **pipeline stages** (e.g., Fetch, Decode, Execute, Writeback).

**Exam Tip:**  Understand how pipelining can improve throughput (the number of instructions completed per unit time) but can also introduce complexities like hazards (e.g., data dependencies).
