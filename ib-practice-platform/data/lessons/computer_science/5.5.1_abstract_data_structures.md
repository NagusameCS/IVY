### Subtopic 1: Introduction to Abstract Data Structures

> **Important:** ADTs provide a way to organize and manage data and related operations, hiding the underlying implementation details. This abstraction allows developers to focus on *what* the data structure does, rather than *how* it does it.

An **abstract data type (ADT)** is a conceptual model that defines a set of operations and their associated behavior without specifying how those operations are implemented. It’s essentially a blueprint for a data structure. ADTs are crucial for modularity, reusability, and efficient software design. 

Consider a `List`.  As an ADT, it defines operations like `add`, `remove`, `get`, and `set`.  Different programming languages and libraries might implement a `List` using an array, a linked list, or another underlying data structure. However, the user of the ADT only needs to know *how* to use the defined operations, not *how* it's physically stored.

**Example:**

Let’s say we want to represent a stack.  As an ADT, a stack defines the operations:
*   `push(item)`: Adds an item to the top of the stack.
*   `pop()`: Removes and returns the item at the top of the stack.
*   `peek()`: Returns the item at the top of the stack without removing it.
*   `isEmpty()`: Checks if the stack is empty.

An implementation of this stack could be using an array or a linked list.  The user of the stack doesn’t care which one is used; they only care about using the `push`, `pop`, `peek`, and `isEmpty` operations.

### Subtopic 2: Stacks

> **Exam Tip:** Stacks are LIFO (Last-In, First-Out) data structures.  Understand the implications of this behavior for algorithm design.

Stacks are linear data structures that follow the **Last-In, First-Out (LIFO)** principle. This means the last element added to the stack is the first one removed.

**Implementation (Conceptual):**

Stacks can be implemented using arrays or linked lists.

*   **Array-based Stack:**  An array is used to store the elements. The top of the stack is usually represented by a pointer/index to the most recently added element.
*   **Linked List-based Stack:**  A linked list is used where each node contains an element and a pointer to the next node. The top of the stack is represented by a pointer to the head of the list.

**Example 1:**

```python
class Stack:
    def __init__(self):
        self.items = []

    def push(self, item):
        self.items.append(item)

    def pop(self):
        if not self.is_empty():
            return self.items.pop()
        else:
            return None  # Or raise an exception

    def peek(self):
        if not self.is_empty():
            return self.items[-1]
        else:
            return None

    def is_empty(self):
        return len(self.items) == 0

# Usage Example
my_stack = Stack()
my_stack.push(1)
my_stack.push(2)
my_stack.push(3)

print(my_stack.pop())  # Output: 3
print(my_stack.pop())  # Output: 2
print(my_stack.peek())  # Output: 1
```

**Example 2:**

Consider a function that evaluates a simple expression like `(2 + 3) * 4`.  A stack can be used to evaluate this expression correctly. The expression is parsed from left to right, and each operand and operator is pushed onto the stack. The operators are applied as they are encountered, and the results are pushed back onto the stack. Finally, the remaining value on the stack is the result of the expression.

### Subtopic 3: Queues

> **Note:** Queues are FIFO (First-In, First-Out) data structures.  This behavior is essential for many real-world applications.

**Queues** are linear data structures that follow the **First-In, First-Out (FIFO)** principle. This means the first element added to the queue is the first one removed.  They are often used to model real-world scenarios like waiting lines or print queues.

**Implementation (Conceptual):**

Like stacks, queues can be implemented using arrays or linked lists.

*   **Array-based Queue:**  An array is used to store the elements.  Care must be taken to manage the front and rear of the queue.
*   **Linked List-based Queue:** A linked list is used where each node contains an element and a pointer to the next node.  The front and rear of the queue are represented by pointers.

**Example 1:**

```python
class Queue:
    def __init__(self):
        self.items = []

    def enqueue(self, item):
        self.items.append(item)

    def dequeue(self):
        if not self.is_empty():
            return self.items.pop(0)
        else:
            return None

    def peek(self):
        if not self.is_empty():
            return self.items[0]
        else:
            return None

    def is_empty(self):
        return len(self.items) == 0

# Usage Example
my_queue = Queue()
my_queue.enqueue(1)
my_queue.enqueue(2)
my_queue.enqueue(3)

print(my_queue.dequeue())  # Output: 1
print(my_queue.dequeue())  # Output: 2
print(my_queue.peek()) # Output: 3
```

### Subtopic 4: Linked Lists

> **Exam Tip:**  Understand the advantages and disadvantages of linked lists compared to arrays (e.g., dynamic resizing, ease of insertion/deletion).

A **linked list** is a linear data structure where elements are not stored in contiguous memory locations. Instead, each element (called a **node**) contains data and a pointer (or link) to the next node in the sequence.

**Key Characteristics:**

*   **Dynamic Size:** Linked lists can grow or shrink dynamically as needed.
*   **Efficient Insertion/Deletion:** Inserting or deleting nodes in the middle of a linked list is generally more efficient than in an array, because you only need to update pointers, rather than shifting elements.

**Example:**

```python
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None

    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
        else:
            current = self.head
            while current.next:
                current = current.next
            current.next = new_node

    def print_list(self):
        current = self.head
        while current:
            print(current.data, end=" ")
            current = current.next
        print()
```

### Subtopic 5:  Applications of ADTs

> **A.1.1 Water**: While seemingly unrelated, the concept of a queue can be applied to modeling a waiting list for water distribution after a drought, ensuring fairness and order of access.

ADTs are fundamental to countless software applications. Some examples include:

*   **Operating Systems:** Queues are used to manage processes and threads.
*   **Databases:** Stacks and queues are utilized in query optimization and transaction management.
*   **Web Servers:** Queues are used to handle incoming requests efficiently.
*   **Game Development:** Stacks and queues are used in pathfinding algorithms and managing game entities.

### Subtopic 6:  ADT vs. Array Implementation

> **Important:**  Understanding the trade-offs between using an ADT and a direct array implementation is crucial for selecting the appropriate approach.

While you *can* implement a stack or queue using a simple array, the ADT approach provides several benefits:

*   **Abstraction:**  The ADT hides the underlying implementation details.
*   **Flexibility:**  You can change the underlying implementation without affecting the code that uses the ADT.
*   **Maintainability:**  The code becomes more modular and easier to maintain.
