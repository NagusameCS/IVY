## Introduction to Programming

## Overview

This lesson introduces fundamental programming concepts, focusing on algorithms, sequencing, and debugging. Students will learn how to translate real-world problems into step-by-step instructions that a computer can execute.

## Key Concepts

*   **Algorithms:** A finite sequence of well-defined instructions for solving a problem or accomplishing a task.
*   **Sequencing:** The order in which instructions are executed in a program.
*   **Debugging:** The process of identifying and correcting errors (bugs) in a program.
*   **Pseudocode:** An informal way to represent an algorithm, using natural language and flowcharts.

### Subtopic 1: What is an Algorithm?

> **Important:** An algorithm is more than just a list of steps; it’s a *solution* to a problem. It must be unambiguous and finite.

An **algorithm** is a set of instructions designed to solve a specific problem. Think of it like a recipe – a recipe provides a sequence of steps to achieve a desired outcome (a delicious dish). Similarly, an algorithm provides a sequence of steps to solve a computer problem.

**Example:** Let's create an algorithm to make a cup of tea.

1.  Boil water.
2.  Place a tea bag in a mug.
3.  Pour the boiled water into the mug.
4.  Let the tea steep for 3 minutes.
5.  Remove the tea bag.
6.  Add milk and sugar to taste.

This is an algorithm! It’s a precise, step-by-step guide to making a cup of tea.  Now let's represent this algorithm in **pseudocode**:

```pseudocode
BEGIN
    INPUT: Water
    INPUT: Tea Bag
    INPUT: Mug
    BOIL Water
    PLACE Tea Bag in Mug
    POUR Boiled Water into Mug
    WAIT 3 minutes
    REMOVE Tea Bag
    INPUT: Milk
    INPUT: Sugar
    ADD Milk to Mug
    ADD Sugar to Mug
    END
```

> **Exam Tip:**  The IB assesses the ability to design and analyze algorithms. You’ll often be asked to describe an algorithm in pseudocode or flowchart format.

**Worked Example 1:**

**Problem:** Write an algorithm to calculate the area of a rectangle.

**Algorithm (Pseudocode):**

```pseudocode
BEGIN
    INPUT: Length of rectangle (L)
    INPUT: Width of rectangle (W)
    Calculate: Area = L * W
    OUTPUT: Area
END
```

**Worked Example 2:**

**Problem:** Write an algorithm to determine if a number is even or odd.

**Algorithm (Pseudocode):**

```pseudocode
BEGIN
    INPUT: Number (N)
    Calculate: Remainder = N % 2  // Modulo operator (%) gives the remainder of a division
    IF Remainder = 0 THEN
        OUTPUT: "Even"
    ELSE
        OUTPUT: "Odd"
    ENDIF
END
```

### Subtopic 2: Sequencing – The Order Matters

> **Note:** The order in which you execute instructions is *crucial*. Changing the order can drastically alter the outcome of a program.

**Sequencing** refers to the order in which instructions are executed in a program. In a sequential program, each instruction is carried out one after another, in the order they are written.

**Example:**  Consider the algorithm for making a sandwich. If you put the bread on the plate *before* adding the filling, you’ll have a messy sandwich.  The correct sequence is:

1.  Get bread.
2.  Get filling.
3.  Place bread on plate.
4.  Add filling to bread.

**Example Python Code (Illustrative):**

```python
print("Get bread")
print("Get filling")
print("Place bread on plate")
print("Add filling to bread")
```

The output of this code would be:

```
Get bread
Get filling
Place bread on plate
Add filling to bread
```

### Subtopic 3: Introduction to Debugging

> **Example:** A common bug is a syntax error – the computer doesn't understand the code because it's not written correctly. Another common bug is a logical error – the code runs, but doesn't produce the desired result.

**Debugging** is the process of finding and fixing errors (bugs) in a program. Bugs can occur for various reasons, such as:

*   **Syntax Errors:**  Mistakes in the way the code is written (e.g., missing a semicolon).
*   **Logical Errors:**  The code runs but doesn't do what the programmer intended.
*   **Runtime Errors:** Errors that occur while the program is running (e.g., dividing by zero).

**Debugging Techniques:**

*   **Read Error Messages:**  Error messages often provide clues about the type and location of the error.
*   **Print Statements:** Inserting `print()` statements to display the value of variables at different points in the program can help identify where the error occurs.
*   **Step-by-Step Execution:**  Some programming environments allow you to step through the code line by line, observing the program's state.

**Worked Example 3:**

**Problem:** The following Python code is supposed to calculate the sum of two numbers, but it's not working correctly.

```python
num1 = 5
num2 = 10
sum = num1 + num2
print("The sum is:" , sum)
```

**Debugging:**

The code might produce the output "The sum is: 15". However, we want to calculate the sum of 5 and 10. There's no bug in the *code*, the output is correct. This illustrates how seemingly incorrect output can be a symptom of a deeper misunderstanding of the problem.

**To demonstrate a bug, imagine the following erroneous code:**

```python
num1 = 5
num2 = "10"  # Intentionally using a string instead of an integer
sum = num1 + num2
print("The sum is:" , sum)
```

**Output:** `TypeError: unsupported operand type(s) for +: 'int' and 'str'`

This error arises because you're attempting to add an integer (5) to a string ("10"). To fix this, you must ensure the data types are compatible.
