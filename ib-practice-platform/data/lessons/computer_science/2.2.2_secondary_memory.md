## Secondary Memory – An Introduction

Secondary memory, also known as *permanent storage*, is a non-volatile storage medium used to retain data for extended periods, even when the power is off. Unlike **primary memory** (RAM), which is used for actively running programs and data, secondary memory provides a long-term archive.  This allows computers to store large amounts of information like operating systems, applications, documents, images, and videos persistently. The crucial difference is volatility: primary memory is lost upon power loss, whereas secondary memory retains data. > **Exam Tip:** Examiners frequently test students’ understanding of storage hierarchies and the trade-offs between speed and capacity.

### Magnetic Storage (HDDs) - Deeper Dive

Let’s delve deeper into **magnetic storage**, the technology underlying HDDs.

1.  **Magnetic Domains:** Data is stored as magnetic domains – regions within the magnetic material that have a north or south magnetic polarity.
2.  **Recording Data:**  A read/write head generates a magnetic field to align these domains, representing 0s and 1s.
3.  **Reading Data:** The read head detects the magnetic field of the aligned domains, converting it back into digital data.
4.  **Tracks and Sectors:** Data is organized on the platters in concentric circles called **tracks**. Each track is divided into sectors, which are the smallest units of storage.
5.  **Rotational Speed (RPM):**  The speed at which the platters rotate (measured in Revolutions Per Minute) significantly affects HDD performance. Higher RPM = faster data access.

    > **Exam Tip:** Understanding the physical limitations of magnetic storage (e.g., mechanical wear, access latency) is crucial for assessing performance bottlenecks in computer systems.



## Worked Example 1

**Question:** A computer system uses an HDD with a rotational speed of 7200 RPM.  Calculate the average time (in milliseconds) it takes to access a single sector on the HDD. Assume a sector size of 512 bytes and a full rotation time of 3.4 milliseconds.

**Solution:**

1.  **Calculate the number of rotations per second:** 7200 RPM = 7200 rotations / 60 seconds = 120 rotations/second.
2.  **Calculate the time for one rotation:** 3.4 milliseconds
3.  **Calculate the time to access one sector:** Since a sector is defined by a specific point on the rotating platter, the time to access it is the time for one full rotation.
4.  **Convert to milliseconds:** 3.4 milliseconds

Therefore, it takes 3.4 milliseconds to access a single sector on the HDD.



## Worked Example 2

**Question:** An SSD has a transfer rate of 550 MB/s. How many seconds does it take to copy a 10 GB file to the SSD?

**Solution:**

1.  **Convert GB to MB:** 10 GB = 10 * 1024 MB = 10240 MB
2.  **Calculate the time:** Time = Data Size / Transfer Rate
3.  **Calculate:** Time = 10240 MB / 550 MB/s = 18.64 seconds (approximately)

Therefore, it takes approximately 18.64 seconds to copy the 10 GB file to the SSD.
