## Connecting Computational Thinking and Program Design

## Overview

This lesson focuses on the crucial link between **computational thinking** and **program design**. Students will explore how core computational thinking concepts – decomposition, pattern recognition, abstraction, and algorithm design – directly influence the process of creating effective and efficient programs.  We’ll move beyond simply writing code to understand *why* we write it a certain way.

## Key Concepts

*   **Computational Thinking as a Foundation:**  Understanding that computational thinking is not just about writing code, but a set of problem-solving skills applicable to any situation.
*   **Mapping Thinking to Design:**  Recognizing how specific computational thinking techniques translate into concrete design choices within a program.
*   **Iterative Design Process:** Implementing a cyclical design process incorporating both problem analysis and solution implementation.

### Subtopic 1: Decomposition – Breaking Down Problems

> **Important:** Decomposition is the process of breaking down a complex problem into smaller, more manageable sub-problems.  This is a fundamental step in both computational thinking and effective program design.

Decomposition is the first pillar of computational thinking. Consider a complex task like planning a road trip.  A direct approach would be to simply start planning the route, but this is unlikely to yield an effective plan.  Instead, we’d decompose the problem into smaller steps:

1.  **Define the Goal:**  Determine the destination and desired travel dates.
2.  **Sub-Problem 1: Route Planning:**  Determine the optimal route using mapping software (decomposition!).
3.  **Sub-Problem 2: Accommodation:**  Research and book hotels/accommodation along the route.
4.  **Sub-Problem 3: Activities:**  Identify points of interest and plan activities at each stop.
5.  **Sub-Problem 4: Logistics:**  Manage transportation, budgeting, and packing.

**Example 1:**

*   **Problem:** Design a program to calculate the area of a rectangle.
*   **Decomposition:**
    *   **Step 1:**  Get the length of the rectangle as input.
    *   **Step 2:**  Get the width of the rectangle as input.
    *   **Step 3:**  Calculate the area (length * width).
    *   **Step 4:**  Display the calculated area.

```python
def calculate_rectangle_area():
    length = float(input("Enter the length of the rectangle: "))
    width = float(input("Enter the width of the rectangle: "))
    area = length * width
    print("The area of the rectangle is:", area)

calculate_rectangle_area()
```

**Example 2:**

*   **Problem:**  Write a function to sort a list of numbers.
*   **Decomposition:**
    *   **Step 1:**  Choose a sorting algorithm (e.g., Bubble Sort, Insertion Sort).
    *   **Step 2:**  Implement the algorithm’s logic – this involves breaking the sorting process into smaller steps, such as comparing adjacent elements and swapping them if they are in the wrong order.
    *   **Step 3:**  Test the function with different lists of numbers.

### Subtopic 2: Pattern Recognition – Identifying Reusable Solutions

> **Note:** Pattern recognition involves noticing similarities in problems and developing general solutions that can be applied across different contexts. This is key to creating efficient algorithms.

Once you've decomposed a problem, **pattern recognition** helps you identify recurring elements that can be formalized into reusable solutions.  Returning to our road trip example, we’ve likely travelled similar routes before. Recognizing this pattern allows us to apply the same planning approach (decomposition) to new trips.

In programming, this manifests as creating **functions** or **modules**. A function that calculates the area of a rectangle can be reused in any program that needs to calculate areas of rectangles.

**Example 1:**

*   **Problem:**  A program needs to perform the same calculations repeatedly – calculating the area and perimeter of various shapes.
*   **Pattern Recognition:**  Recognize that the calculations involve basic geometric formulas.
*   **Solution:** Create a library of functions that implement these formulas (area_rectangle(), perimeter_rectangle(), etc.). This avoids redundant code.

### Subtopic 3: Abstraction – Hiding Complexity

> **Example:**  Consider using a library like `math` in Python. You don’t need to understand the underlying implementation of `sqrt()` to use it. You simply call the function with a number and get the square root.

**Abstraction** is the process of hiding unnecessary complexity and focusing on essential details.  In program design, this involves using **high-level constructs** (like functions, libraries, or APIs) that represent complex operations without requiring the programmer to understand the internal workings.

This reduces code complexity, improves maintainability, and allows developers to focus on the overall logic of the program.

**Example 1:** Using a database query language (like SQL) – you don’t need to understand the low-level details of how the database stores and retrieves data; you simply write the query.

### Subtopic 4: Algorithm Design – Creating Step-by-Step Solutions

> **Exam Tip:**  IB often tests your ability to describe algorithms in pseudocode or flowcharts.  Focus on clearly outlining the steps involved, independent of any specific programming language.

Algorithm design is the process of creating a **step-by-step procedure** to solve a problem. It’s the culmination of the other computational thinking concepts – decomposition allows us to break down the problem, pattern recognition helps us identify reusable solutions, and abstraction hides unnecessary complexity.

**Example 1:**  Write an algorithm to find the largest number in a list.

1.  **Initialization:** Set a variable `largest` to the first element in the list.
2.  **Iteration:** Iterate through the remaining elements of the list.
3.  **Comparison:** For each element, compare it with the current value of `largest`.
4.  **Update:** If the current element is greater than `largest`, update `largest` to the current element.
5.  **Return:** After iterating through all elements, return the value of `largest`.
