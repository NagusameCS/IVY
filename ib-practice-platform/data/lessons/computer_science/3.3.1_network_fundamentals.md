## 3.3.1 - Networks: Network Fundamentals

### 1. Introduction to Networks

> **Important:** A **network** is a collection of interconnected computing devices that can exchange data. Networks allow devices to share resources, communicate, and access information. *Networks are crucial for nearly all modern computing.*

Networks come in various sizes and complexities. We'll start with the basic concepts:

*   **Nodes:** Any device connected to a network (e.g., computers, servers, smartphones).
*   **Links:** The physical or wireless connections that allow nodes to communicate.
*   **Data Transmission:** The process of sending information between nodes.



**Example 1:** Consider a simple home network. Your laptop, smartphone, and smart TV are nodes, connected by a Wi-Fi router (a node itself). The Wi-Fi is the link, and the data you send from your laptop to your smartphone is transmitted across this network.

### 2. Network Topologies

The arrangement of nodes and links in a network is called its **topology**. Different topologies offer varying levels of redundancy and performance.

*   **Bus Topology:** All devices are connected to a single cable (the *backbone*). Simple but vulnerable to failure – a break in the cable disrupts the entire network.
*   **Star Topology:** All devices are connected to a central hub or switch. More reliable than bus topology, as a failure of one device doesn't affect the rest.
*   **Ring Topology:** Devices are connected in a closed loop. Data travels in one direction.  Less common in modern networks.
*   **Mesh Topology:** Devices are interconnected with multiple pathways. Offers high redundancy and reliability but is complex and expensive.

> **Exam Tip:**  The IB exam frequently tests understanding of network topologies and their advantages/disadvantages. Be prepared to draw diagrams and explain how data flows through different topologies.

**Example 2:**  Imagine a small office. A *star topology* is often chosen because if one computer fails, the rest of the network continues to operate. A *bus topology* might be used in a very small home network.



### 3. Network Protocols

**Protocols** are a set of rules that govern how devices communicate on a network. They ensure that data is transmitted and received correctly. Some key protocols include:

*   **TCP/IP (Transmission Control Protocol/Internet Protocol):** The foundation of the Internet. TCP handles reliable data transmission, while IP handles addressing and routing.
*   **HTTP (Hypertext Transfer Protocol):** Used for transferring web pages and other web content.
*   **FTP (File Transfer Protocol):** Used for transferring files between computers.
*   **SMTP (Simple Mail Transfer Protocol):** Used for sending email.

> **Note:** Protocols work at different layers of the **OSI model** (Open Systems Interconnection model), which provides a conceptual framework for network communication. *Understanding the OSI model is helpful but not always directly tested on the IB exam.*



### 4. IP Addressing

Every device connected to a network needs a unique **IP address** (Internet Protocol address) to be identified and located.

*   **IPv4:** The most common addressing scheme, using 32-bit addresses (e.g., 192.168.1.1).
*   **IPv6:** A newer addressing scheme using 128-bit addresses, designed to accommodate the increasing number of devices connected to the Internet.


**Example 3:**  Your router assigns your laptop a specific IPv4 address (e.g., 192.168.1.100). This allows other devices on the network to send data to your laptop.



### 5. Network Hardware

Several hardware components are essential for building and operating networks:

*   **Hubs:** Simple devices that broadcast data to all connected devices (inefficient).
*   **Switches:** More intelligent devices that forward data only to the intended destination device, improving network performance.
*   **Routers:** Connect different networks together (e.g., your home network to the Internet).
*   **Network Interface Card (NIC):** A hardware component that allows a device to connect to a network.

### 6.  Network Security Basics

> **Important:** Network security is crucial for protecting data and devices from unauthorized access.  Concepts like firewalls and encryption are important.

*   **Firewall:** A barrier that controls network traffic, blocking unauthorized access.
*   **Encryption:** Converting data into an unreadable format to protect its confidentiality.
